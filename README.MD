## What is d3m0n?

d3m0n is an AI agent for offensive security work. You describe a task in natural language ‚ÄĒ the agent decides which tools to call, executes them on your system, reasons over the real output, and continues until the task is complete.

It is not a wrapper that fakes tool output. When you ask it to scan a target, it actually runs `nmap`. When you ask it to enumerate subdomains, it actually runs `amass`. Every tool call is logged to SQLite with full arguments and output.

Built in Go ‚ÄĒ compiles to a single binary with no runtime dependencies.

---

## Features

- **Autonomous agent loop** ‚ÄĒ model calls tools, reasons over real output, chains multiple tools automatically
- **22 integrated security tools** ‚ÄĒ from recon to exploitation to reporting
- **CVE lookup** ‚ÄĒ searches NVD database for vulnerabilities in discovered services
- **Target memory** ‚ÄĒ agent saves and recalls information about targets across sessions
- **Professional report generation** ‚ÄĒ generates Markdown and dark-themed HTML pentest reports from conversation history
- **Session logging** ‚ÄĒ every tool call stored in SQLite with full args and output
- **Config-driven** ‚ÄĒ model, timeout, safe mode, allowed paths all set in `config.yaml`
- **Local LLM support** ‚ÄĒ works with Ollama, LM Studio, or any OpenAI-compatible endpoint
- **Safe mode** ‚ÄĒ path traversal protection, restricts file operations to configured directories
- **Streaming output** ‚ÄĒ responses stream token by token, re-rendered with glamour markdown
- **Beautiful CLI** ‚ÄĒ lipgloss styled UI, spinners, rendered markdown, slash commands

## Configuration

Copy `config.yaml.example` to `config.yaml`:

```yaml
# LLM
model: "anthropic/claude-3.5-haiku"     # any OpenRouter model
base_url: "https://openrouter.ai/api/v1"
api_key: "your-api-key-here"            # or set OPENROUTER_API_KEY env var

# Agent
max_history: 20      # messages kept in context window
timeout: 300         # seconds per request before timeout

# Security
safe_mode: true      # restrict file ops to allowed_paths only
allowed_paths:
  - .
  - ./output
  - /tmp

# Prompt
system_prompt: "You are a professional cybersecurity red team expert."
```

`config.yaml` is gitignored ‚ÄĒ your keys stay local.

### Local LLM

```yaml
# Ollama
model: "llama3.2"
base_url: "http://localhost:11434/v1"
api_key: "ollama"

# LM Studio
model: "local-model"
base_url: "http://localhost:1234/v1"
api_key: "lm-studio"
```

> Tool calling requires a capable model. Llama 3.2, Mistral 7B+, and Qwen2.5 work well. Models under 7B parameters are unreliable for agentic tasks. Avoid free-tier OpenRouter models (`:free` suffix) ‚ÄĒ they hallucinate tool outputs.

## Session Logging

Every session is stored in `data/agent.db` (SQLite ‚ÄĒ no server required):

```bash
# list recent sessions
sqlite3 data/agent.db \
  "SELECT id, start_time FROM sessions ORDER BY start_time DESC LIMIT 10;"

# view tool calls from a session
sqlite3 data/agent.db \
  "SELECT tool, args FROM tool_logs WHERE session_id='20260601-143022';"

# view all remembered targets
sqlite3 data/agent.db "SELECT host, info, updated FROM targets;"
```

Or use the `/history` and `/targets` slash commands inside d3m0n.

## Security

- **Safe mode** restricts all file read/write to `allowed_paths` in config ‚ÄĒ path traversal is blocked
- **API key** loaded from `config.yaml` or `OPENROUTER_API_KEY` env var ‚ÄĒ never hardcoded
- **`config.yaml` is gitignored** ‚ÄĒ credentials stay local
- All tool execution happens locally on your machine
- Session data and reports stored locally in `data/` and `output/`

> d3m0n is intended for authorized security testing only. Only use against systems you own or have explicit written permission to test. The author is not responsible for misuse.
